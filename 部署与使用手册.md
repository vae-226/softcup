# 部署与使用手册

## 1. 产品简介

灵境导游是一套面向景区场景的 AI 数字人智能导览系统。本版本已改为原生微信小程序形态，实现游客导览首页和智慧中枢管理后台，重点展示数字人导览、本地景区知识问答、个性化路线推荐、景区百科、周边服务、知识库管理、数字人配置和游客感受度分析。

旧 Web 原型已移动到 `备份/`，根目录为微信小程序工程。

## 2. 运行环境

- 操作系统：Windows / macOS / Linux
- 开发工具：微信开发者工具
- AppID：可使用测试号，或将 `project.config.json` 中的占位 AppID 替换为团队自己的小程序 AppID
- 后端服务：当前首版不依赖后端，问答和路线推荐使用本地数据

## 3. 导入与运行

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本仓库根目录。
4. AppID 使用测试号或团队自己的 AppID。
5. 点击“编译”，进入 `pages/guide/guide` 首页。
6. 点击底部“管理”，进入 `pages/admin/admin` 智慧中枢管理后台。

## 4. 功能使用说明

### 4.1 数字人导览首页

首页提供以下能力：

- 展示灵山胜境 AI 数字人导游“灵灵”
- 展示导游对话气泡和声波动效
- 支持文本输入提问
- 支持快捷问题按钮
- 展示历史文化、自然风光、亲子家庭三类推荐路线
- 展示景区百科磁贴
- 展示洗手间、素斋、停车场、游客中心等服务入口
- 展示底部讲解控制条和底部导航

建议演示流程：

1. 打开小程序首页。
2. 查看数字人欢迎讲解。
3. 在输入框输入“灵山大佛有什么特色？”并发送。
4. 点击亲子家庭路线的“开始游览”。
5. 观察对话内容、当前讲解标题和路线推荐变化。

### 4.2 智慧中枢管理后台

管理后台提供以下能力：

- 顶部概览：今日服务人次、本周服务人次、平均响应时间、游客满意度、问答准确率
- 数字人形象管理：姓名、服饰、音色、讲解风格和保存配置入口
- 游客感受度报告：情绪趋势、游客关注点、服务优化建议
- 知识库管理：知识条目、分类、关联景点、启用状态和编辑/停用入口
- 热门问答与反馈：星级、情绪标签、反馈内容和建议处理动作

管理后台视觉参考 `stitch/stitch_lingjing_guide_admin/` 的 Stitch 设计稿。

### 4.3 当前版本边界

语音输入、TTS、口型同步和数字人形象为前端演示效果。真实落地时，不应把大模型 API Key 写入小程序前端，应通过 HTTPS 后端代理或微信云函数调用模型服务。

## 5. 测试

本地服务逻辑可用 Node 测试：

```bash
node --test tests/guide-service.test.mjs
```

当前测试覆盖：

1. 游客询问灵山大佛时获得相关景点讲解。
2. 游客选择亲子兴趣时推荐亲子家庭路线。
3. 管理后台提供运营概览、知识库和反馈数据。

## 6. 后续升级建议

1. 通过 HTTPS 后端或微信云函数接入 DashScope / Qwen 等大模型接口。
2. 接入微信录音能力和 ASR 服务，实现真实语音输入。
3. 接入 TTS 和数字人 SDK，实现真实口型同步和表情驱动。
4. 将 `data/scenic.js` 升级为数据库和向量知识库。
5. 增加 GPS、二维码点位或蓝牙 Beacon，实现景区点位触发讲解。
