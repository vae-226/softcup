# 部署与使用手册

## 1. 产品简介

灵境导游是一套面向景区场景的 AI 数字人智能导览系统。本版本已改为原生微信小程序形态，首版实现游客导览首页，重点展示数字人导览、本地景区知识问答、个性化路线推荐、景区百科和周边服务入口。

旧 Web 原型已移动到 `备份/`，根目录为微信小程序工程。

## 2. 运行环境

- 操作系统：Windows / macOS / Linux
- 开发工具：微信开发者工具
- AppID：可使用测试号，或将 `project.config.json` 中的占位 AppID 替换为团队自己的小程序 AppID
- 后端服务：当前首版不依赖后端，问答和路线推荐使用本地数据

## 3. 导入与运行

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本仓库根目录。
4. AppID 使用测试号或团队自己的 AppID。
5. 点击“编译”，进入 `pages/guide/guide` 首页。

## 4. 功能使用说明

### 4.1 数字人导览首页

首页提供以下能力：

- 展示灵山胜境 AI 数字人导游“灵灵”
- 展示导游对话气泡和声波动效
- 支持文本输入提问
- 支持快捷问题按钮
- 展示历史文化、自然风光、亲子家庭三类推荐路线
- 展示景区百科磁贴
- 展示洗手间、素斋、停车场、游客中心等服务入口
- 展示底部讲解控制条和底部导航

建议演示流程：

1. 打开小程序首页。
2. 查看数字人欢迎讲解。
3. 在输入框输入“灵山大佛有什么特色？”并发送。
4. 点击亲子家庭路线的“开始游览”。
5. 观察对话内容、当前讲解标题和路线推荐变化。

### 4.2 当前版本边界

当前首版只迁移游客导览首页，管理后台页暂未迁移。后台能力仍在总体设计文档中保留，后续可新增 `pages/admin` 页面实现知识库管理、数字人配置、游客感受度报告和数据大屏。

语音输入、TTS、口型同步和数字人形象为前端演示效果。真实落地时，不应把大模型 API Key 写入小程序前端，应通过 HTTPS 后端代理或微信云函数调用模型服务。

## 5. 测试

本地服务逻辑可用 Node 测试：

```bash
node --test tests/guide-service.test.mjs
```

当前测试覆盖：

1. 游客询问灵山大佛时获得相关景点讲解。
2. 游客选择亲子兴趣时推荐亲子家庭路线。

## 6. 后续升级建议

1. 新增管理后台小程序页面或独立 Web 管理台。
2. 通过 HTTPS 后端或微信云函数接入 DashScope / Qwen 等大模型接口。
3. 接入微信录音能力和 ASR 服务，实现真实语音输入。
4. 接入 TTS 和数字人 SDK，实现真实口型同步和表情驱动。
5. 将 `data/scenic.js` 升级为数据库和向量知识库。
6. 增加 GPS、二维码点位或蓝牙 Beacon，实现景区点位触发讲解。
