# 总体设计文档

## 1. 项目概述

### 1.1 项目名称

灵境导游 - AI数字人景区导览系统

### 1.2 建设目标

构建一个具备多模态交互能力的 AI 数字人导游系统，为游客提供实时问答、个性化路线讲解和情感互动，并为景区管理者提供游客反馈分析和运营数据支持。

### 1.3 业务痛点

当前景区导览服务存在以下问题：

1. 高峰期导游资源不足，游客难以及时获取高质量讲解。
2. 传统录音导览内容固定，不能回应个性化问题。
3. 缺少陪伴感和情感交互，沉浸体验弱。
4. 管理方难以量化游客反馈，运营优化缺乏数据支撑。

## 2. 总体方案

系统分为游客交互侧和管理后台侧。

### 2.1 游客交互侧

- 支持文本输入和语音输入
- 数字人使用语音、表情、口型同步回应
- 提供景区知识问答和景点讲解
- 根据游客兴趣推荐游览路线

### 2.2 管理后台侧

- 管理知识库内容
- 管理数字人形象、服装、声音和角色风格
- 分析游客交互记录，生成情感趋势和建议报告
- 展示运营数据大屏

## 3. 系统架构设计

## 3.1 逻辑架构

系统分为四层：

1. 表现层  
   游客端 App / H5 页面、景区终端、管理后台 Web 页面。

2. 能力层  
   语音识别、语音合成、多模态大模型、数字人驱动引擎。

3. 数据层  
   景区知识库、游客交互日志、反馈数据、数字人配置数据。

4. 管理层  
   知识库运营、服务监控、满意度分析、可视化报表。

## 3.2 推荐技术路线

- 多模态大模型：Qwen-Omni / GPT-4o / GLM-4.1V
- 语音识别：Whisper / FunASR / Paraformer
- 语音合成：Edge-TTS / ChatTTS / 商业 TTS
- 数字人驱动：Live2D / WebRTC 数字人 SDK / Unity 或 Unreal
- 后端：FastAPI / Flask / Spring Boot
- 数据库：MySQL + Redis + 向量数据库

## 4. 核心功能设计

### 4.1 智能问答

基于景区知识库构建 RAG 检索增强问答流程：

1. 将景区资料进行分块
2. 提取向量并建立索引
3. 接收游客问题后进行召回与重排
4. 将检索结果与问题一起输入大模型生成回答
5. 返回文本、语音和数字人驱动指令

### 4.2 个性化路线推荐

系统根据游客兴趣标签、游览时长、是否亲子、当前时段等信息，动态匹配最佳游览路线，生成讲解重点和停留建议。

### 4.3 情感互动

系统通过语音语调、用词风格和基础情感识别结果，控制数字人的表情和动作状态，提升陪伴感和自然度。

### 4.4 游客感受度分析

通过对交互文本、满意度反馈和热点问题进行统计，输出：

- 游客关注点分析
- 情感趋势报告
- 服务优化建议

## 5. 非功能设计

### 5.1 性能目标

- 语音问答平均延迟控制在 5 秒以内
- 事实问答准确率目标不低于 90%
- 系统高峰时段稳定运行，无长时间无响应

### 5.2 可扩展性

- 模块化设计，方便替换模型、知识库和数字人引擎
- 可平滑扩展多景区、多角色、多语言能力

### 5.3 弱网与定位复杂场景

可扩展以下策略：

1. 常见问题和基础讲解离线缓存
2. GPS 信号弱时结合蓝牙 Beacon / 二维码点位触发
3. 对重点区域使用边缘推理与本地语音缓存

## 6. 原型实现说明

当前提交版本为静态演示原型，具备以下特点：

- 无需复杂部署即可运行
- 覆盖比赛核心功能模块
- 可用于初赛演示和视频录制
- 已预留真实工程化改造接口
